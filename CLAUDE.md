# CLAUDE.md

Guidance for Claude Code working in this repo.

## What this is

`student-quiz` — браузерное PWA для самопроверки студентов (тренировка к
собеседованию КОРД/НПУ). Не «софтовый продукт» — это маленькое статичное
приложение + база тестов в JSON. Архитектура **A**: статичный SPA (Vite + TS,
vanilla, без фреймворка) + nginx + каталог `tests/` как volume. **Серверного
кода нет.** Подробности — `docs/superpowers/specs/2026-06-27-student-quiz-design.md`.

## Команды

```
npm test            # юнит-тесты core (vitest) — гонять после правок core
npm run build       # tsc --noEmit + vite build
npm run dev         # дев-сервер (отдаёт /tests/* из каталога tests/)
docker compose up -d --build   # пересборка+деплой (нужно при изменении КОДА)
```

Node 20+ (asdf; `.tool-versions`). Деплой крутится на `mac-mini-artem:8089`,
доступ из tailnet по Tailscale.

## Архитектура / код

- `src/core/` (чистый TS, покрыт тестами): `loader` (грузит+валидирует базу),
  `grader` (балл/разбор), `shuffle` (Фишер–Йетс, `buildOrder`). Правишь core →
  обнови/прогони тесты.
- `src/ui/`: `screens.ts` (рендер-функции, возвращают HTML-строки) + `app.ts`
  (контроллер: меню → прогон → результат).
- **Прогресс не хранится** — состояние прохождения только в памяти (`Run`),
  localStorage НЕ используется. Не возвращай его без явной просьбы.
- Модель: тесты хранятся раздельно (`LoadedQuiz.tests`), меню тем.
- **5 типов вопросов** (поле `type`), все «всё-или-ничего»:
  - `single` — radio, ровно 1 верный (`options`/`correct`);
  - `multi` — checkbox, точное совпадение множества;
  - `cloze` — пропуск в тексте (`___`), выбор слова; грейдится как single;
  - `order` — `items[]` + `correct` (правильный порядок id); UI ▲▼;
  - `match` — `left[]`/`right[]`/`pairs` (left→right); UI select на строку;
    классификация = `match`, где несколько `left` указывают на один `right`.
  Добавляя тип, правь loader (валидация) + grader (isCorrect) + shuffle
  (`choiceIds`) + screens.renderQuestion + app.showQuestion (сбор ответа).

## База тестов (`tests/`)

- `manifest.json`: `version` (косметика), `title`, `debug`, `tests[]` (порядок в
  меню). `test-XX.json`: `id`, `title`, `questions[]`.
- **`id` вопроса уникален во ВСЕЙ базе** — loader падает на дубликатах. Префиксы
  по темам (p/m/g/l/s/w/b/e/f/a/o/d/z/n…).
- Менять базу = править JSON в `tests/`. Данные подхватываются **live** (volume +
  `no-store`), без пересборки. Версию бампать не обязательно (прогресс не хранится).
- Валидация перед деплоем: открыть приложение (битый JSON → понятная ошибка) или
  прогнать правила loader скриптом.

## Дебаг-флаг

`"debug": true` в manifest → id вопроса, экран «Усі питання», кнопка «Скинути».
**Дефолт в репозитории — `false`.** На сервере включают подменой файла:
`sed -i '' 's/"debug": false/"debug": true/' tests/manifest.json` (live).
Поэтому в рабочем дереве `tests/manifest.json` часто помечен изменённым
(`debug:true`) — это оперативное состояние, **не коммить** этот флип.

## Контент

- Источник — iCloud, папка
  `~/Library/Mobile Documents/com~apple~CloudDocs/20-29 Обучение/25 Полиция и оружие/`
  (выверенная шпаргалка `Співбесіда УПО`, `Картка_Поліція_мобільна`, docx ТТХ,
  плакаты, `Стрельбы Лекции`). Эти материалы пользователь считает достоверными.
- **Весь контент тестов — на украинском.** Правильные ответы брать дословно из
  источника; неверные — правдоподобные из той же фактуры.
- «Сложные» вопросы (тест `Визначення дослівно`): верный вариант — полное
  определение слово-в-слово, дистракторы — то же с точечными искажениями
  (заменено/убрано/добавлено одно слово/оборот).
- Гранатомёты (РПГ-7, лекция #6) — намеренно вне базы.

## Гочи

- **Bind-mount после git-операций:** переключение веток/stash, затрагивающее
  `tests/`, может «оторвать» volume контейнера (nginx начнёт отдавать 404 на
  `/tests/*`). Лечится `docker compose up -d --force-recreate`.
- Контейнер собирает образ с базой по умолчанию (COPY tests), но volume её
  перекрывает — на рантайме источник правды это каталог `tests/` на хосте.

## Процесс

Фичи/контент — обычно на ветке, потом merge в `main`. После каждой заметной
правки пользователь сам прокликивает приложение (телефон+комп) и даёт «ок» —
это приёмочный гейт; в дебаге для просмотра всех вопросов есть «Усі питання».
